A probability problem

I have two real numbers x1 and x2. You choose one of them xi to know. Then, you guess xi⩾x3-i or the other way around. Is there a strategy to win strictly more often than 50%? If so, state and prove it; if not, prove it.

Yes. There are at least two methods:

  • Choose a number s∈ℝ at random as the standard (e.g. by normal distribution). If xi⩾s, answer “larger”; otherwise, answer “smaller”.

  • Assign every real number x a probability p(x), where p is strictly increasing (e.g. let p(x)≜e-x1+e-x). Answer “larger” with probability p(xi).

A subtle problem

Is there a C++ pure function int f(int), such that f(f(x)) == x holds for any int x? Prove it.

This problem is taken from Pascal Bourguignon's Software. If there are bignums as in Scheme or Haskell, we could just take

f(x) = { 0, x=0 x+1, x>0∧x≡1⁡(mod⁡2) -x+1, x>0∧x≡0⁡(mod⁡2) x-1, x<0∧x≡1⁡(mod⁡2) -x-1, x<0∧x≡0⁡(mod⁡2) ..

Every four number forms a cycle, except 0. The first subtlety is that we are not dealing with ℤ, but a finite subset of it. Another subtlety lies in the encoding of int. According to the standard, the encoding scheme is not specified. So the perfect answer is “don't know”. A good answer is “no”, because 2's complement is the de facto standard.

A np-complete problem

Given a set of segments of integer lengths, you should put them on the positive axis without overlapping. Each segment must have the coordinate of its left endpoint to be divisible by a given number (which might be different for each segment). Can you find the smallest length to hold all the segments in polynomial time (assuming p≠np)? If so, give an algorithm; if not, prove it and give a heuristic algorithm anyway.

This problem is inspired by alignment in memory layout. We can reduce the subset-sum problem to it.

A weak language

We define a variant of Brainfuck called Brainfuck–. Now we loop the same number of times as the value in the current memory cell the moment we encounter [.

  1. Write a Brainfuck– program which ends with ∑i=1n in the first memory cell if that cell was initialized with n.

  2. Is Brainfuck– Turing-complete?

  3. Can you write a Brainfuck– interpreter in Brainfuck–? If so, briefly explain your thought; if not, prove it.

  1. [>+<]>[-[<+>]]

    It shows that Brainfuck– can express some nontrivial programs. I think some students are mislead.

  2. No. All programs terminate.

  3. No. We can't write a meta-interpreter in any (sufficiently expressive) total language. Suppose we have an interpreter p, then ∀q,x,⟦p⟧(q,x)=⟦q⟧(x). Let ⟦e⟧(x)≜⟦p⟧(x,x)+1, then

    ⟦p⟧(e,e) = ⟦e⟧(e) = ⟦p⟧(e,e)+1.

    A contradiction.
